# 🏛️ Architecture - Clinic Management System

این مستند به تشریح معماری سیستم مدیریت درمانگاه می‌پردازد که با استفاده از Node.js، MongoDB و React پیاده‌سازی شده است. الگوی معماری اصلی پروژه **MVC (Model - View - Controller)** بوده و تمام اجزای سیستم بر اساس اصل جداسازی مسئولیت‌ها طراحی شده‌اند.

---

## 📐 1. Overview (نمای کلی)

سامانه مدیریت درمانگاه شامل دو بخش اصلی است:

- **Backend (سرور):** پیاده‌سازی APIها، منطق تجاری، مدل داده‌ها و ارتباط با پایگاه داده.
- **Frontend (کاربر):** رابط کاربری مبتنی بر ReactJS برای استفاده بیماران، پزشکان و مدیران.
- **Database (پایگاه‌داده):** MongoDB برای ذخیره‌سازی داده‌ها با مدل‌های Mongoose.

---

## 🧱 2. معماری لایه‌ای (MVC Pattern)

### 🧩 Model (مدل‌ها)
- پیاده‌سازی با استفاده از **Mongoose** (ODM برای MongoDB)
- تعریف ساختار داده‌ها مانند: کاربران، پزشکان، بیماران، نوبت‌ها، نسخه‌ها، داروها، درخواست‌ها و نظرات

📁 مسیر: `/backend/models/`

---

### 🔁 Controller (کنترلرها)
- دریافت درخواست‌های HTTP از روترها
- اعتبارسنجی داده‌ها و فراخوانی سرویس‌ها
- مدیریت خطاها و ارسال پاسخ مناسب

📁 مسیر: `/backend/controllers/`

---

### 🧠 Service Layer (لایه سرویس)
- شامل منطق تجاری مستقل از HTTP
- مدیریت پیچیدگی عملیات‌ها مانند ثبت نوبت، صدور نسخه، بررسی موجودی دارو و ...
- قابل استفاده مجدد در تست‌ها و کنترلرها

📁 مسیر: `/backend/services/`

---

### 🔐 Middleware
- مدیریت توکن JWT
- احراز هویت و کنترل سطح دسترسی کاربران بر اساس نقش
- لاگ‌گیری و هندل کردن خطاهای عمومی

📁 مسیر: `/backend/middlewares/`

---

### 🛣️ Routes (مسیرهای API)
- تعریف مسیرهای RESTful برای کاربران، پزشکان، نوبت‌دهی، داروخانه و...
- نگاشت مسیرها به کنترلرهای مرتبط

📁 مسیر: `/backend/routes/`

---

### ⚙️ Config & Utils
- تنظیمات اتصال به پایگاه‌داده، پورت سرور، کلیدهای امنیتی و ...
- توابع کمکی مانند رمزنگاری، ساخت ID یکتا، ارسال پیامک یا ایمیل

📁 مسیر: `/backend/config.js`, `/backend/utils/`

---

## 🖼️ 3. Frontend Architecture (معماری فرانت‌اند)

### ساختار اصلی React:

- **Pages:** صفحات مختلف مانند ثبت‌نام، نوبت‌ها، پنل مدیریت، نسخه‌ها و فروشگاه دارو
- **Components:** اجزای قابل استفاده مجدد مانند فرم ورود، جدول نوبت‌ها، کارت دارو
- **State Management:** استفاده از Context API یا Redux برای مدیریت وضعیت‌ها
- **Routing:** استفاده از React Router برای مسیرهای مختلف پنل‌ها

📁 مسیر: `/frontend/`

---

## 🗄️ 4. Database Design (MongoDB)

- هر کلکشن با استفاده از **Schema‌های Mongoose** تعریف شده
- روابط با استفاده از reference (`ObjectId`) بین کلکشن‌ها برقرار شده
- ساختار ERD در فایل [ERD.png](./ERD.png) آورده شده است

---

## 🔐 5. Security Architecture

- **JWT Authentication:** برای احراز هویت کاربران پس از ورود
- **RBAC (Role Based Access Control):** سطح دسترسی کاربران طبق نقش (بیمار، پزشک، مدیر درمانگاه، ناظر و ...)
- **Hashing:** رمزنگاری پسوردها با bcrypt
- **Validation:** اعتبارسنجی ورودی‌ها با express-validator

---

## 🚀 6. Deployment

- محیط توسعه: Node.js + Nodemon
- محیط تولید: Node.js با Reverse Proxy (مثل Nginx) یا استفاده از Docker
- متغیرهای محیطی در `.env` نگهداری می‌شود

---

## 🧪 7. Testing

- **Backend:** با استفاده از Jest و Supertest برای تست API
- **Frontend:** با React Testing Library و Mock Service Worker (MSW)

📁 مسیر تست‌ها: `/tests/`

---

## 🔄 8. API Communication Flow

[Frontend] → [Routes] → [Middleware] → [Controller] → [Service] → [Model (MongoDB)]
